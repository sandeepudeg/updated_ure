# MVP Requirements Document

## Introduction

This document specifies the **Minimum Viable Product (MVP)** requirements for implementing essential privacy and security features for the GramSetu agricultural assistant application. This MVP focuses on delivering core privacy capabilities that provide immediate value while maintaining system stability and minimizing deployment complexity.

The MVP prioritizes:
1. **Amazon Cognito** for anonymous authentication (replacing localStorage IDs)
2. **Session-based privacy** with automatic data deletion via DynamoDB TTL
3. **IP address hashing** to protect user location privacy
4. **Basic encryption** at rest using AWS-managed keys

**Deferred to Future Releases:**
- VPC isolation (Requirement 3) - Complex infrastructure change
- Advanced audit logging (Requirement 7) - Beyond MVP scope
- GDPR compliance features (Requirement 12) - Regulatory requirements
- Custom KMS keys (Requirement 5.4) - AWS-managed keys sufficient for MVP

## Glossary

- **GramSetu_System**: The complete agricultural assistant application including all AWS services
- **Cognito_Identity_Pool**: Amazon Cognito service component that provides temporary AWS credentials for anonymous users
- **Session_Manager**: Component responsible for managing user sessions with configurable time-to-live
- **Lambda_Function**: AWS Lambda serverless compute function (ure-mvp-handler)
- **Conversation_Store**: DynamoDB table (ure-conversations) storing user conversation history
- **User_Profile_Store**: DynamoDB table (ure-user-profiles) storing user profile information
- **Data_Bucket**: S3 bucket (ure-mvp-data-us-east-1-188238313375) for application data storage
- **Legacy_User_ID**: Existing user identifier stored in browser localStorage
- **Cognito_Identity_ID**: Unique identifier generated by Cognito Identity Pool
- **TTL_Attribute**: DynamoDB Time-To-Live attribute for automatic record deletion
- **IP_Hash**: SHA-256 hashed IP address for privacy protection

## MVP Requirements

### MVP Requirement 1: Cognito Anonymous Authentication (Core)

**User Story:** As a farmer using GramSetu, I want to access the application anonymously without registration, so that I can maintain my privacy while receiving agricultural assistance.

**Priority:** MUST HAVE (P0)

#### Acceptance Criteria

1. THE Cognito_Identity_Pool SHALL provide unauthenticated access for anonymous users
2. WHEN an anonymous user accesses GramSetu_System, THE Cognito_Identity_Pool SHALL generate a Cognito_Identity_ID within 2 seconds
3. WHEN a Cognito_Identity_ID is generated, THE GramSetu_System SHALL issue temporary AWS credentials valid for 60 minutes
4. THE GramSetu_System SHALL use Cognito_Identity_ID as the primary user identifier for all data operations
5. WHEN credential expiration approaches within 5 minutes, THE GramSetu_System SHALL automatically refresh the temporary credentials

**MVP Simplifications:**
- No rate limiting per IP (deferred to post-MVP)
- Basic error handling only
- No advanced credential management

---

### MVP Requirement 2: Legacy User Migration (Core)

**User Story:** As an existing user, I want my data preserved during the authentication upgrade, so that I can continue using GramSetu without disruption.

**Priority:** MUST HAVE (P0)

#### Acceptance Criteria

1. WHEN a user with Legacy_User_ID accesses GramSetu_System after upgrade, THE GramSetu_System SHALL automatically create a Cognito_Identity_ID
2. THE GramSetu_System SHALL migrate all data associated with Legacy_User_ID to the new Cognito_Identity_ID
3. THE GramSetu_System SHALL maintain the user's conversation history during migration
4. WHEN migration completes, THE GramSetu_System SHALL update the browser storage with the new Cognito_Identity_ID
5. THE GramSetu_System SHALL support both Legacy_User_ID and Cognito_Identity_ID authentication for 30 days (reduced from 90 days)

**MVP Simplifications:**
- Simplified migration logic (no retry mechanism)
- 30-day compatibility window instead of 90 days
- No in-app migration notifications

---

### MVP Requirement 3: Session-Based Auto-Deletion (Core)

**User Story:** As a privacy-conscious user, I want my conversation data to be automatically deleted after my session ends, so that my agricultural queries are not retained indefinitely.

**Priority:** MUST HAVE (P0)

#### Acceptance Criteria

1. THE Conversation_Store SHALL include a TTL_Attribute set to 3 hours from last activity
2. THE User_Profile_Store SHALL include a TTL_Attribute set to 3 hours from last activity
3. WHEN a TTL_Attribute expires, THE DynamoDB service SHALL automatically delete the record within 48 hours
4. WHEN a user interacts with GramSetu_System, THE GramSetu_System SHALL update the TTL_Attribute to extend the session

**MVP Simplifications:**
- Fixed 3-hour session duration (no configuration)
- No explicit session termination API
- No session tokens (use Cognito_Identity_ID directly)
- Rely on DynamoDB's automatic TTL deletion

---

### MVP Requirement 4: IP Address Hashing (Core)

**User Story:** As a user, I want my IP address to be protected, so that my location cannot be tracked through network identifiers.

**Priority:** MUST HAVE (P0)

#### Acceptance Criteria

1. WHEN GramSetu_System receives a user request containing an IP address, THE GramSetu_System SHALL apply SHA-256 hashing to the IP address before any storage operation
2. THE GramSetu_System SHALL append a fixed salt value to IP addresses before hashing
3. THE GramSetu_System SHALL store the salt value as an environment variable in Lambda configuration
4. THE GramSetu_System SHALL not store plaintext IP addresses in any data store

**MVP Simplifications:**
- Fixed salt value (no rotation)
- Salt stored in Lambda environment variables (not Secrets Manager)
- No rate limiting based on hashed IP

---

### MVP Requirement 5: Basic Encryption at Rest (Core)

**User Story:** As a compliance officer, I want all stored data encrypted at rest, so that data breaches cannot expose sensitive user information.

**Priority:** MUST HAVE (P0)

#### Acceptance Criteria

1. THE Conversation_Store SHALL enable encryption at rest using AWS-managed KMS keys
2. THE User_Profile_Store SHALL enable encryption at rest using AWS-managed KMS keys
3. THE Data_Bucket SHALL enable default encryption using AWS-managed KMS keys

**MVP Simplifications:**
- AWS-managed keys only (no customer-managed keys)
- No key rotation (AWS handles automatically)
- No encryption failure handling beyond default AWS behavior

---

### MVP Requirement 6: HTTPS Enforcement (Core)

**User Story:** As a user, I want my data encrypted during transmission, so that my conversations cannot be intercepted by third parties.

**Priority:** MUST HAVE (P0)

#### Acceptance Criteria

1. THE GramSetu_System SHALL enforce TLS 1.2 or higher for all API Gateway endpoints
2. THE CDN SHALL enforce HTTPS for all content delivery
3. THE GramSetu_System SHALL reject HTTP requests and redirect to HTTPS

**MVP Simplifications:**
- Use existing CloudFront and API Gateway TLS configuration
- No custom cipher suite configuration
- No certificate management (use AWS-managed certificates)

---

### MVP Requirement 7: Basic Performance (Core)

**User Story:** As a user, I want the privacy features to not degrade application performance, so that I can receive timely agricultural assistance.

**Priority:** MUST HAVE (P0)

#### Acceptance Criteria

1. WHEN Cognito authentication is enabled, THE Lambda_Function SHALL maintain response time below 3 seconds for 95% of requests
2. WHEN encryption is applied, THE GramSetu_System SHALL maintain response time below 3 seconds for 95% of requests
3. THE Cognito_Identity_Pool SHALL maintain 99.9% availability

**MVP Simplifications:**
- No connection pooling optimization
- No DNS caching
- Basic performance monitoring only

---

### MVP Requirement 8: Simple Deployment (Core)

**User Story:** As a DevOps engineer, I want the MVP privacy features to be easily deployable, so that I can roll out the changes quickly.

**Priority:** MUST HAVE (P0)

#### Acceptance Criteria

1. THE GramSetu_System SHALL provide a Python deployment script for Cognito_Identity_Pool creation
2. THE GramSetu_System SHALL provide a Python script to enable DynamoDB TTL on existing tables
3. THE GramSetu_System SHALL provide a Python script to enable S3 and DynamoDB encryption
4. THE GramSetu_System SHALL document all required IAM permissions
5. THE deployment scripts SHALL validate AWS credentials before making changes

**MVP Simplifications:**
- No Infrastructure as Code templates (CloudFormation/Terraform)
- No rollback procedures
- No monitoring dashboards
- Manual configuration validation

---

### MVP Requirement 9: Basic Data Privacy Controls (Optional)

**User Story:** As a privacy-conscious user, I want basic controls over my data, so that I can delete my information if needed.

**Priority:** SHOULD HAVE (P1)

#### Acceptance Criteria

1. THE GramSetu_System SHALL provide a data deletion API endpoint for immediate user data removal
2. WHEN a user requests data deletion, THE GramSetu_System SHALL delete all associated records within 5 minutes
3. THE GramSetu_System SHALL collect only data required for providing agricultural assistance

**MVP Simplifications:**
- No data export functionality
- No PII flagging
- No anonymization for analytics
- Simple deletion without confirmation workflow

## Deferred Requirements (Post-MVP)

The following requirements from the comprehensive spec are **explicitly deferred** to future releases:

### Deferred: VPC Isolation (Requirement 3)
**Reason:** Complex infrastructure change requiring VPC setup, NAT Gateway, VPC Endpoints, and extensive testing. High cost and complexity for MVP.

**Future Implementation:** Phase 2 (after MVP validation)

### Deferred: Advanced Audit Logging (Requirement 7)
**Reason:** Comprehensive logging adds complexity and cost. Basic CloudWatch logs sufficient for MVP.

**Future Implementation:** Phase 2 (security hardening)

### Deferred: GDPR Compliance Features (Requirement 12)
**Reason:** Full regulatory compliance requires legal review, documentation, and breach procedures beyond MVP scope.

**Future Implementation:** Phase 3 (compliance certification)

### Deferred: Custom KMS Keys (Requirement 5.4)
**Reason:** AWS-managed keys provide adequate security for MVP. Customer-managed keys add operational overhead.

**Future Implementation:** Phase 2 (if required by security audit)

### Deferred: Advanced Session Management (Requirement 2 - partial)
**Reason:** Session tokens, configurable durations, and explicit termination add complexity. Fixed 3-hour TTL sufficient for MVP.

**Future Implementation:** Phase 2 (user experience enhancement)

### Deferred: IP Salt Rotation (Requirement 4.4)
**Reason:** Automatic salt rotation requires Secrets Manager and rotation Lambda. Fixed salt adequate for MVP.

**Future Implementation:** Phase 2 (security hardening)

### Deferred: Data Export API (Requirement 8.6-8.7)
**Reason:** Data portability is important but not critical for MVP. Deletion capability is higher priority.

**Future Implementation:** Phase 2 (GDPR compliance)

## MVP Success Criteria

The MVP is considered successful when:

1. ✅ Users can access GramSetu anonymously using Cognito (no localStorage IDs)
2. ✅ Existing users are automatically migrated without data loss
3. ✅ Conversation data auto-deletes after 3 hours of inactivity
4. ✅ IP addresses are hashed before storage
5. ✅ All data encrypted at rest using AWS-managed keys
6. ✅ HTTPS enforced for all traffic
7. ✅ Response time remains below 3 seconds for 95% of requests
8. ✅ Deployment completed in under 2 hours
9. ✅ Zero data loss during migration
10. ✅ AWS cost increase below 10% (vs. 20% for full implementation)

4. Verify data migrated to Cognito ID
5. Verify conversation history preserved

2. Wait 3 hours + 48 hours
3. Verify data deleted from DynamoDB

2. Check DynamoDB records
3. Verify IP is hashed (not plaintext)

### Test 5: Encryption at Rest
1. Check DynamoDB table settings
2. Verify encryption enabled
3. Check S3 bucket settings
4. Verify default encryption enabled

1. Run load test with 100 concurrent users
2. Measure response times
3. Verify 95th percentile < 3 seconds

## Conclusion

This MVP delivers core privacy and security capabilities with minimal complexity and cost. It provides immediate value through anonymous authentication, automatic data deletion, and basic encryption while deferring complex features like VPC isolation and advanced compliance to future phases.

The MVP can be deployed in 2 weeks with minimal risk and cost, providing a solid foundation for future security enhancements.
